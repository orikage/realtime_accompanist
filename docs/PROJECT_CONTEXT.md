# Project Context

## 背景

ユーザーは、ハッカソンやMaker Faireに出せるようなリアルタイム音楽プロダクトを検討している。
発想の核は「右手でフレーズを弾いていたら、自動的にキーやコードを認識して伴奏してくれるプロダクト」。

既存製品としては以下に近いが、完全一致するものは少ない。

- Microsoft Songsmith / MySong: 歌った単音メロディに自動コード・伴奏を付ける。録音/作曲支援寄り。
- DigiTech TRIO / TRIO+: ギター演奏を聴いてベース・ドラムを生成。コード/リズムを教える前提。
- CASIO Chordana Composer / Trackformer: 短いメロディから曲生成。リアルタイム追従ではない。
- Humtap: 鼻歌とタップから曲生成。スマホAI作曲寄り。
- Google Magenta A.I. Duet: 鍵盤入力にAIが応答。伴奏/コード推定ではなくコール＆レスポンス寄り。
- Yamaha/KORGのアレンジャーキーボード: コードを押さえると伴奏。メロディのみから伴奏するわけではない。

## 差別化

既存の自動伴奏は基本的に「コード入力」が必要。
このプロダクトは「右手メロディだけ」からキー/コード候補を推定し、伴奏者が空気を読むようにリアルタイムで追従する。

## 重要な現実認識

単音メロディからコードは一意に決まらない。
たとえば E という音は C, Am, Em, Fmaj7, Dm9 など多数のコードに含まれる。
そのため、「正しいコードを当てる」ではなく「自然な伴奏を生成する」ことを成功条件にする。

## 展示での価値

Maker Faireでは、音だけでなく「AIが何を考えているか」が見えるUIが重要。
推定キー、コード候補、確信度、次コード候補、ジャンル、伴奏密度が表示されると体験が伝わりやすい。
